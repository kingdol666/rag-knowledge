Features

Eight Pillars of Knowledge Intelligence

From raw PDF to structured knowledge graph — one unified pipeline.

📄

Document Parsing

PDF / Word / Excel / PPT / images → Markdown via MinerU OCR engine. Structured extraction with tables and figures preserved.

🧠

QDCVR Retrieval

Query-Driven, Content-Verified Retrieval with independent 0–8 content scoring. Vectors are fast. Content is accurate.

🔍

Multi-Strategy Search

BM25 + vector two-stage recall. Cross-KB enterprise search. Tag semantic + graph expansion. balance_kbs for fair distribution.

📊

Knowledge Graph

Neo4j-powered graph with 14 tools. Entity/relation graphs, cross-KB document bridges, centrality discovery, path queries.

💡

Experience Library

E0–E12 lifecycle management. Structured problem→solution→lessons. P0/P1/P2 credibility tiers. Stale detection & decay cycles.

🔌

76 MCP Tools

KB CRUD, search, graph, experience, parsing, tags, vector/index, service lifecycle. All MCP-native, any client works.

🎯

14 Claude Code Skills

Natural-language commands with bilingual triggers (中/EN). Auto-dispatch to Archival agent. Self-contained for global plugin.

🤫

Silent Headless

Every launcher runs with zero terminal windows in both dev and prod. Logs unified across ragctl, Tauri, and MCP.

Quick Start

Three Commands to Launch

From zero to a fully working platform. No manual dependency hell.

Step 1 — Clone
# Clone the repository git clone https://github.com/kingdol666/rag-knowledge.git cd rag-knowledge
Step 2 — One-Click Setup
# Installs ALL deps + models (uv, Python 3.12, torch, BGE-M3...) ./ragctl setup
Step 3 — Start Everything
# Start all services (silent, zero terminal windows) ./ragctl up # → Backend: http://localhost:8765 # → Web UI: http://localhost:6789

Why

Why This Project

What makes RAG Knowledge Platform different from traditional tools.

Traditional KB Tools

  • Separate search, storage, and AI layers
  • Manual setup with complex CLI
  • Hard to integrate with agents
  • Separate dev/prod configurations
  • Terminal windows clutter
  • No structured knowledge reuse
  • Single KB search strategy

RAG Knowledge Platform

  • Unified: parse → index → search → graph → experience
  • One command or one phrase: ragctl setup
  • Native: 76 MCP tools + 14 skills, any client
  • Single config.yml with runtime --appmode switch
  • Zero terminal windows in both dev and prod
  • E0–E12 experience lifecycle with credibility tiers
  • Multi-strategy: BM25 + vector + graph + enterprise
